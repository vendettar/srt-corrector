# 改进记录

- CLI 用法现为 `python3 -m srt_corrector ...`；如果有 README/脚本引用旧用法需更新。
- `SRTEntry.original_text` 默认为 `None` 后回填原值；若要保持不可变，可改为只读或通过工厂函数设置。
- `parse_srt` 解析依赖空行分隔；可添加预清理（去 BOM、统一换行）提升健壮性。
- 模糊匹配阈值、窗口步长/倍率当前硬编码；可集中成配置或通过 CLI 暴露以便调优。
- CLI 仍是手写参数解析；可换 `argparse`/类型校验，改进错误提示与帮助。
- 模糊匹配微调有双重循环，长文本可能较慢；可限制窗口或加调试日志开关；建议补充基准/单测防回归。
